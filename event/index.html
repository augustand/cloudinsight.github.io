事件流

事件流显示 Cloudinsight 产品内部产生的所有事件。目前包括以下几类事件：

  • 报警事件
    • 报警触发
    • 报警关闭
    • NO DATA
  • 系统事件
    • 策略禁用 / 解除禁用
    • 策略增 / 删 / 改
    • Cloudinsight Agent 启动

报警事件

报警触发

显示报警触发条件，以及最近一次触发时间。

  • 更新时间：策略再次被触发，从而更新该事件的时间。
  • 创建时间：策略第一次被触发，继而创建该事件的时间。

点击报警状态或编辑报警，进入相应页面。

报警关闭

显示报警触发条件，和报警关闭时间。以及与之对应的报警触发事件的，最后一次策略被触发的时间。

  • 创建时间：策略由触发状态恢复正常，继而创建该事件的时间。

NO DATA

在 Cloudinsight，NO DATA 也被独立为一个策略，记录它是否被触发。然后当 NO DATA 恢复，即有数据点时，不会产生 NO DATA 关闭事件。

是因为，当有数据点产生时，Cloudinsight 会记录当前是 OK 或 ALERT，所以不需要重复提醒。

  • 更新时间：策略再次被触发，从而更新该事件的时间。
  • 创建时间：策略第一次被触发，继而创建该事件的时间。

系统事件

策略禁用 / 解除禁用

显示禁用的执行人，以及策略创建人。

点击策略名称，进入状态页查看详情。

策略增 / 删 / 改

所有策略的增删改操作，会产生事件来提醒用户。在修改时，Cloudinsight 会告知修改的内容。

Cloudinsight Agent 启动

当 Cloudinsight Agent 在主机中启动，会生成事件。并点击仪表盘，查看该主机的运行情况。
