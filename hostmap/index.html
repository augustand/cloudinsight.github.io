Hostmap

Hostmap 就是把你监控的所有主机，所有平台服务都放在一张图表上，可以对其任意进行分组，筛选，根据颜色来区分使用健康状况，是非宕机。通过 Hostmap 可以轻松把物理机，各种云服务器放在一起，进行各种维度的比较，把监控的基础组件的运行情况都放在一起进行展示。

基本操作

Hostmap 分组是根据 tag 进行的，现在有2种加 tag 的方式，一种是服务器配置好监控后，在 system 上面添加各种 key:value 的 tag 标签。

另一种是在 Cloud Insight Agent 配置文件里面配置，例如在 mysql 配置文件里面加上2个tag：

tags:
  - region: beijing

在这之后就可以点击平台，拓补，默认会出现所有的平台列表。目前 Hostmap 可以通过过滤器过滤数据，也可以通过 tag 对数据进行分组。

过滤器

Hostmap 通过过滤器会剩下符合条件的应用，目前有3种过滤维度：

主机状态（up/down）：过滤出目前运行／不运行的的主机

平台服务列表（apache/docker等）：过滤出监控此平台服务的主机，例如添加 apache，就显示目前监控 apache 的所有主机

tag(bd:mysql等)：过滤出有这个 tag 标签的主机，例如添加 bd:mysql，就显示出所有有 bd:mysql 标签的主机

以上所有过滤条件均支持多选，例如可以 docker apache，也可以 docker bd:mysql 等，鼠标放上去，会显示所有可选项。

通过 tag 进行分组

Hostmap 通过 tag 对所有主机进行分组，前面在每个主机上都添加了 tag，接下来就通过不同 key，相同 value 的会自动聚合成一组，通过这个可以很方便的管理不同地区，不同人员，不同系统等维度的数据进行分组。鼠标放上去，会显示所有可选的 key。
