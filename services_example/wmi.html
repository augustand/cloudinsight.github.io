WMI

WMI 是一项核心的 Windows 管理技术；用户可以使用 WMI 管理本地和远程计算机。

Cloudinsight 通过收集 WMI 数据来实时可视化 Windows 应用程序／服务的性能，以及显示和其它应用联合使用时情况。

配置 WMI 监控

1. 配置需要收集的指标

配置 conf.d/wmi_check.yaml ，将需要收集的指标添加到配置文件中 。

instances:

  # Fetch the number of processes and users
  - class: Win32_OperatingSystem
    metrics:
      -  [NumberOfProcesses, system.proc.count, gauge]
      -  [NumberOfUsers, system.users.count, gauge]


  - class: Win32_PerfFormattedData_PerfProc_Process
    metrics:
      - [ThreadCount, ciagent.threads.count, gauge]
      - [VirtualBytes, ciagent.mem.virtual, gauge]
    filters:
      - Name: ciagent
    constant_tags:
      - 'role:test'

3. 重启探针

通过探针管理器重启探针 Actions，restart。

4. 确认状态

在探针管理器中查看信息栏 Log and Status 下面的 Agent Status，如果有以下信息的输出，代表配置正确。

Checks
======

  [...]

  iis
  ---
      - instance #0 [OK]
      - Collected 8 metrics & 0 events

有关 Agent Info 信息的查看，请访问帮助中心，查看 Cloudinsight Agent 常用操作。
