常见问题

我们列举一些用户的常见问题，希望能够帮助您解决问题。

端口占用

Cloud Insight Agent 使用 10010 端口发送数据，所以若该端口被占用，则会报错。

error: [Error 98] Address already in use

您可以修改其他程序的端口，或者修改 oneapm-ci-agent.conf 配置文件，来指定 Cloud Insight Agent 端口号。

listen_port:17121

编码格式

Cloud Insight 只支持 utf-8 的编码格式，其他 zh_CN 的编码格式会报错。

http_emitter [ERROR]: TypeError('expected string or buffer',)

解决办法为，修改机器编码为 zh_CN.UTF-8。

Hostname 无效

安装完探针后，默认的 hostname 为 localhost 等一系列无效的 hostname。此时需修改 oneapm-ci-agent.conf 中 hostname 为有效值（不含下划线等特殊字符）。

配置文件缩进问题

执行 configcheck时，出现以下错误时，则代表配置文件有语法错误。

mysql.yaml contains errors:
    mapping values are not allowed in this context
   in "<byte string>", line 5, column 11
Fix the invalid yaml files above in order to start the OneAPM CI Agent.
A useful external tool for yaml parsing can be found at
http://yaml-online-parser.appspot.com/

此时，请参考 conf.d/平台服务.yaml.example 中配置示例，进行修改。

instances:
  - server: localhost
    user: my_username
    pass: my_password

instances 前不接任何字符，从属于 instances 的配置 - server 前接 2 个空格。user 必须和 server 对齐。
